# 오퍼레이터 위키 레퍼런스 (밸런스 패치용)

나무위키 성능 섹션을 압축한 밸패 참고 자료. 각 파일 = `위키 수치` + `🎮 현재 게임 구현(레버)`.
재능/스킬을 위키대로 검증·재조정할 때 사용한다.

- 레버 정의: `app/game/data/operator-passives.ts` (PassiveSpec)
- 궁극 종류/에너지: `app/game/data/operator-ult-energy.ts` (ULT_KIND / ULT_ENERGY / ULT_ARTS_CONSUME)
- 전투 로직: `app/game/hooks/useRunState.ts`
- 전투 시스템 문서(물리 이상·2단계 아츠): `test/battle-wiki.txt`
- 밸런스 sim: `node_modules/.bin/tsx test/sim.ts matrix gear buff` → 목표 47~49% (21~22/45)
- **전원 명함(잠재 0) 기준**: 모든 오퍼 = 정예화 최종 재능 + 잠재능력 0. 잠재(1돌~5돌)는 미구현(위키에 "1잠 필수"여도 명함 유지)
- **재구성 메모**: physAnomaly 스킬 한정 = `anomalySkills`(operators.ts) / 카드 표시문 = `SKILL_EFFECT_LINES`(cards.ts) / 연계 조건부 = "데미지 항상 + 효과만 조건부"(BattleState.lastLinkOp)

## 저장된 오퍼
※ `endministrator~arclight`(아래 8종)은 위키 원문 배율표 미보유 → **현재 구현 기반 복원**(스킬 설명은 게임 데이터=위키 입력분). 나머지는 위키 원문 압축.
※ **갱신(2026-06-30)**: `endministrator`·`lifeng`·`chenqianyu`은 **위키 원문 배율표 전체 입수** → 각 파일 하단 "📊 위키 원문 배율표"에 Lv1~12 전수 저장(복원분 아님). DD류 리뉴얼 구현분도 반영.
